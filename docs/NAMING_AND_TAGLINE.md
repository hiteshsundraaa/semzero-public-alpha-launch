# Naming and tagline

`SemZero` does not immediately explain the product by itself. The tagline must do that work consistently.

## Primary tagline

```text
PR review for hidden assumptions in dbt changes.
```

## Result-oriented tagline

```text
See the hidden assumption, blast radius, and replay evidence before merge.
```

## Current-version positioning

```text
SemZero catches dbt PRs that pass tests but silently change downstream SQL assumptions.
```

## How to explain the name if asked

`Sem` points to semantic meaning. `Zero` points to a zero-trust posture toward implicit assumptions in changes.

Do not over-explain the name in the README. Lead with the tagline and the demo result.

## Naming rule

Until the product has external proof, do not rename. Focus on precision, demo quality, and real-repo validation.
