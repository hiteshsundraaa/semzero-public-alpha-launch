# Troubleshooting SemZero Setup

## `dbt manifest not found`

SemZero needs `target/manifest.json`.

Fix:

```bash
dbt deps
dbt compile
```

Or pass a custom manifest path:

```bash
semzero assumption-ci --dbt-manifest path/to/manifest.json --base-ref origin/main
```

## GitHub Action runs but finds no dbt files

SemZero only reviews dbt-like changes by default:

- `models/**/*.sql`
- `models/**/*.yml`
- `models/**/*.yaml`
- `macros/**/*.sql`
- `snapshots/**/*.sql`
- `seeds/**/*`
- `dbt_project.yml`

Check the generated `changed_files.txt` artifact.

## PR comment is missing

Check that the workflow has:

```yaml
permissions:
  contents: read
  pull-requests: write
```

Also check the uploaded `comment.md` artifact. If it exists but the comment did not post, the issue is usually GitHub token permissions.

## Replay Lite did not run

Replay Lite is optional. Add:

```text
.semzero/replay_fixtures.json
```

Use `.semzero/replay_fixtures.example.json` as the starting template generated by:

```bash
semzero init-assumption-ci --output-dir .
```

## Too many warnings

Keep SemZero in shadow mode and record feedback:

```bash
semzero assumption-feedback \
  --receipt data/semzero_assumption_gate/receipt.json \
  --stable-finding-id <id-from-comment> \
  --disposition false_positive \
  --comment "Too broad for this model"
```

Then use:

```bash
semzero assumption-precision-eval --receipt-dir data/semzero_assumption_gate
```

## Check setup quickly

```bash
semzero doctor-assumption-ci --repo .
```
